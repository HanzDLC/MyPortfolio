Chat Trigger → AI Agent to Base44

An end-to-end n8n automation where a chat trigger initiates an AI agent that processes prompts and pushes structured data directly into a Base44 chatbox application.

Problem

As QA testers performing manual testing, our workflow involved multiple manual handoffs that created bottlenecks. After identifying bugs during testing, we would:

  • Write an issue description
  • Use a custom GPT in ChatGPT to get a formatted output
  • Send the formatted output via Messenger to our supervisor
  • Wait for the supervisor to manually post it to Base44

This multi-step process delayed system changes after bug reporting and created unnecessary dependencies on supervisor availability.

Tools Used
n8n Workflow Automation DeepSeek v3.2 AI Model OpenRouter API Base44 Internal API Chat Trigger HTTP Requests Session Management
Workflow
1
QA tester submits bug description via chat trigger interface
2
AI Agent (DeepSeek v3.2) processes the raw input using custom prompt engineering to format the bug report into structured developer-ready documentation
3
System extracts formatted output and prepares Base44 session identifiers
4
Validates Google authentication session with Base44 API to ensure access is still active
5
If session is valid, builds HTTP payload with formatted message content
6
Posts formatted bug report directly to Base44 chatbox, bypassing the manual Messenger handoff
Error Handling
Google Session Validation - Checks authentication status before attempting to post to Base44 to prevent failed submissions
Conditional Logic Flow - IF node ensures workflow only proceeds when session validation returns a valid user identity
Payload Construction Safeguards - Code node builds structured JSON payload with proper headers and authentication tokens before API call
Fallback Message Extraction - Multiple fallback paths in code to extract AI output from different possible response structures

Custom GPT → AI Agent to Google Sheets

A custom GPT routes formatted QA bug reports through an n8n AI agent, which parses and appends structured rows directly into Google Sheets for streamlined bug tracking.

Problem

As QA testers for a Management Information System (MIS), we needed to log bug reports and issues into structured Excel sheets with specific formatting requirements. Our manual workflow involved:

  • Identifying bugs during QA testing
  • Writing issue descriptions
  • Sending each prompt to ChatGPT to format according to our Excel template
  • Copying the ChatGPT output
  • Manually pasting formatted data into Excel/Google Sheets

This repetitive manual process was time-consuming and created bottlenecks in our bug reporting workflow, delaying issue tracking and resolution.

Tools Used
n8n Workflow Automation Custom GPT (ChatGPT) DeepSeek v3.2 AI Model Google Sheets API OpenRouter API JSON Data Processing OAuth 2.0 Authentication
Workflow
1
QA tester submits raw bug description to Custom GPT via ChatGPT interface
2
Custom GPT formats the bug report according to predefined Excel/Google Sheets template structure
3
Formatted output is automatically sent to n8n webhook trigger
4
n8n AI Agent (DeepSeek v3.2) validates and structures the data into individual fields (Issue ID, Description, Severity, Status, etc.)
5
Structured data is mapped to Google Sheets columns and authenticated via OAuth 2.0
6
Data is appended as a new row to the MIS bug tracking Google Sheet, eliminating manual copy-paste
Error Handling
Schema Validation - AI Agent verifies all required fields are present before attempting Google Sheets insertion
OAuth Token Refresh - Automatic token renewal for Google Sheets API to prevent authentication failures
Data Type Enforcement - Ensures numeric fields, dates, and categorical values match Google Sheets column formats
Retry Logic - Automatically retries failed Google Sheets API calls with exponential backoff for rate limiting
